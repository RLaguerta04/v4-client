Overview Principles and token summary for Media Meter v4
🎨
Design Tokens
Single source of truth for color, spacing, radius, and typography. All tokens are CSS custom properties prefixed with --.
🔤
Inter + Inter Tight
Inter for body copy and UI labels. Inter Tight for headings, display numbers, and KPI values with tight letter-spacing.
⚡
Purposeful Motion
Four easing tiers — fast (12ms), base (200ms), slow (300ms), emphasis (280ms) — keep interactions snappy and expressive.

Tooltips

All hover tooltips use a single site-wide system via #md-tooltip. This is the source of truth — do not create new CSS pseudo-element tooltips or separate tooltip divs.

The Rule
✅ Use #md-tooltip
Two modes via _makeTip():
Cursor-following (no arrow) — bar charts, entity pills, rich data.
Anchored (arrow below element) — set label field; tooltip centers below the element with an upward arrow. Use for icon labels.
✅ Keep #ss-pop
Anchored popovers on KPI cards only. Has arrow because it's fixed to a specific element. Do not use for new cases.
✅ Keep .b-tip
Inline rich tooltip on Story Value KPI only. Has arrow. Scoped to one element. Do not use for new cases.
❌ Never add new CSS [data-tip]::after or pseudo-element tooltips — they cannot be styled consistently, overflow the viewport, and can't render rich content.
